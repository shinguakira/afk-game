# 仕様書: *Idle Engineer*（仮）〜 残業の勇者 〜

IT企業を舞台にした放置型RPG。`Melvor Idle` のコアループ（1つの作業を選ぶ→自動で回る→XP/資源が貯まる→オフラインでも進む）を土台に、**題材をIT業務に丸ごと置き換える**。木こり・採掘ではなく「コードを書く・設計する・案件を倒す」。

> 既存エンジン（`client/src/game/`）はデータ駆動。スキル/アイテム/アクション/モンスターは `data/` の設定で定義する。本仕様はその構造に乗る形で書く。**v1 = 今のエンジンのリスキン + α**、それ以降を [将来拡張](#9-将来拡張ロードマップ) に分ける。

# ⭐ ゲームデザイン: 目的と流れ（現行の北極星）

> このセクションが「いま目指している形」。以降の §1〜 は個別メカニクスの詳細・経緯。
> 部下/チームは設計上は最終的に必要だが現状ノイズなので一旦実装削除済み（[起業メタ階層](#d-起業メタ階層終わらない終盤) で再登場予定）。

## 0-1. コアファンタジーと2本の軸
**「無名のコーダーが、技術を極め・金を稼ぎ、起業して伝説になる」**
プレイの動機は **2本の柱** が車の両輪:

| 軸 | 何を伸ばす | 代表的な目標 |
|---|---|---|
| **A. 技術探求** | 言語/フレームワーク/概念/資格を極める | ポリグロット、フルスタック、資格コンプ、全Lv99 |
| **B. 金稼ぎ** | 円・資産・収入を増やす | 初収入 → 月収UP → 年収1000万 → 億り人 → 資産10億 |

技術と金は相互に効く好循環:
`技術↑ → 運用収入↑ → 円 → 設備/装備 → 効率↑ → 技術がさらに速く`。
起業（プレステージ）で両方をリセットし、永続倍率を得て次の周回へ。

## 0-2. 目的の3層（idleの定石）
| 層 | スパン | 目的 |
|---|---|---|
| 短期 | 数十秒〜 | 次のLv/解禁（FW・概念・レシピ・案件・PCティア） |
| 中期 | 1セッション | 昇進・装備更新・スタック習得・**初の継続収入** |
| 長期 | 横断 | 起業メタ進行・資産マイルストーン・称号・コンプ(100%) |

## 0-3. 全体ループ
```
[生産] 言語を書く → コミット                                      ……(軸A:技術)
   │   └ 傘下を深掘り: 概念 / ライブラリ / フレームワーク / OSS / 資格
   │   └ FW実装・インフラ運用 = コミット消費 → 経験値(言語＋領域)に変換
   ▼
[制作] 料理・PC組み立て・はんだづけ → 飯 / PC / 基板（物理制作のみ）
   ▼
[経済] 売却 + 案件報酬 + 運用収入(放置) → 円 ──────────┐    ……(軸B:金)
   │      └ 運用収入 = 実力(主要スキルLv等)から導出する¥/sec。リリース動作も成果物も無い。
   │      └ 円の使い道: 食事・装備・PCパーツ・食材・(将来)事業    │
   ▼                                                              │ (設備で全部速く)
[遂行] 案件(戦闘) → 円・報酬・現場力XP                            │
   ▼                                                              │
[出世] 総合熟練度 → 昇進 → 職種/難案件/新システム解禁 ◀──────────┘
   ▼
[メタ] 起業(プレステージ) → ストック → 永続強化 → 上位段階で新システム解禁
```

## 0-4. 「方向」を生む4つの仕組み

**技術(A)と金(B)のマイルストーンを交互に**織り込む:

1. 初コミット〔技術〕
2. 初売却で初収入〔金〕（まずはフリーランスで単発案件）
3. 初フレームワーク実装〔技術〕
4. 初案件クリア＋初報酬〔金〕
5. **会社に所属 → 月給＋会社ダンジョン解禁**〔金×案件〕★
6. ジュニア昇進
7. **運用収入が発生（実力が放置中も円を生む）**〔金〕★
8. 自作PC（エントリー）〔技術×金〕
9. 職種を選ぶ（ミドル）
10. 1スタック制覇（例: Web系を一定Lv）〔技術〕
11. 最初の会社を制覇（ダンジョン全クリア）〔案件〕
12. シニア昇進＋資格1つ〔技術〕
13. 資産1000万 / 月収◯◯〔金〕
14. テックリード
15. **3大目標へ突入**（[0-10](#0-10-最初の大目標3つのいずれかを達成)）: 🌙月を買う / 🏢でかい会社制覇 / 🧪新言語開発 → いずれか達成で章クリア→起業

> データ: `roadmap.ts`（id / 条件(達成判定) / 報酬 / 次の指示文）。UIは常時「次の目標」を1枚表示＋達成でトースト。
> 3大目標は終盤の分岐ゴール。それ以前は技術↔金↔案件を交互に踏ませて土台を作る。

### B. 解禁の可視化

## 0-5. 金稼ぎ軸（軸B）の具体
- **円の稼ぎ方**
  1. 売却: コミット/不要装備/制作物（基礎収入）
  2. 案件報酬（メイン能動収入）
  3. **運用収入＝継続収入(放置)**: 積み上げた実力が自動で生む ¥/sec ← 金軸の肝
     - **モノを作らない**。「リリース」という操作も、サービス/資産という持ち物も増えない。
     - `mrr = f(主要スキルLv合計, 累積案件クリア, 役職/起業)` の純粋関数。tick/offline 同式で `gold += mrr*dt`。
     - 品質＝実力。技術が高いほど運用収入が高い、という形で自然に効く。
  4. （将来）事業・投資: 起業後の大型収入
- **資産マイルストーン**（金版ロードマップ）: 初任給 → 月収UP → 年収1000万 → 貯金1億 → 資産10億 → 🌙月
- **円のシンク**: **生活費（常時）**・食事・PCパーツ・装備・食材・（将来）オフィス設備・事業投資
- 技術↔金の結合: 技術が高いほど運用収入↑→設備→技術が速く（好循環）。新しい名詞を増やさずに回す。

## 0-6. 生活費・赤字・行動制限（金軸の常時プレッシャー）
- **生活費**: 時間経過で円が**継続的に減る**（家賃・食費ベース等）。放置でも出ていくので、**収入が無いと貯金が削れる**。
  - これにより「フリーランス（単発でムラ）」vs「会社所属（**月給で安定**）」の選択に意味が出る。
- **残高はマイナス可（借金/赤字）**。**ゲームオーバーにはしない**（idleで詰みは作らない）。
- **赤字時のペナルティ（確定: 全体に効く）**:
  - **全作業＆案件の効率ダウン**（生産/制作/戦闘すべて遅くなる）
  - **メンタル上限↓**（戦闘にも波及）
  - ショップ購入不可・案件報酬の一部天引き
  - 赤字が深いほど重い。**詰みはしない**が抜けるまでツラい → 収入確保（案件/運用収入/就職）へ強く誘導。
- 生活費は段階で上がる（生活水準/役職に応じて）。**稼ぎが増えると支出も増える**緊張感。

## 0-7. 「100%」の位置づけ
全スキルLv99／全資格／全案件／図鑑コンプ = やり込みの終着点の**1つ**。
③称号 と ④起業メタ と ⑤資産 があるので、**100%が唯一のゴールではない**。

## 0-8. 実装順（軽い→重い・効果大→）
1. **A. キャリア・ロードマップ**（最優先・背骨）
2. **B. 解禁の可視化**（軽い・即効）
3. **運用収入＝継続収入(放置)**（金軸の肝・実力から導出）＋資産マイルストーン
4. **C. 称号・実績**
5. **D. 起業メタ階層**（大物・ここで部下/チーム復活）

## 0-9. 世界モデル: 案件=モンスター / 会社=ダンジョン
- **案件 = モンスター**。倒す＝納品。
- **案件達成は技術XPも入る**: 各案件に**関連スタック**（言語/フレームワーク）が紐づき、クリアで現場力XPに加えて**その言語/技術のXP**が入る。
  → 「React案件」を回すと React/TS が伸びる等、**案件(軸)↔技術(軸)が直結**。生産で書くだけでなく“実戦で育つ”ルートになる。
- **会社 = ダンジョン**。会社は**案件グループ（順序付き＋最後に役員/上司ボス）**を保有する。全クリア＝**その会社を制覇**。
- **入社 = 面接官バトル ＋ スキル要件**:
  - その会社の**入社要件（特定言語/総合の最低Lv 等）**を満たすのが前提（門前払い）。
  - 要件を満たしたら**面接官（=案件モンスターの一種）を倒す**＝内定。技術質問/ホワイトボードコーディング等のギミック持ちボス（[5.x](#5-敵--案件上司戦闘デザイン)）。
- **所属の自由**:
  - **フリーランス / アルバイト（無所属）**: いつでも**単発案件**を受けられる（常時供給・報酬控えめ・自由・面接不要）。
  - **会社所属**: その会社の**固有案件（ダンジョン）**にアクセスでき、**月給（継続収入＝金軸に直結）**が入る。転職可（再面接）。
- データ概形（実装時）:
  ```ts
  Monster { ..., relatedSkills?: { skill, xp }[] }          // 案件→技術XP
  Company { id, name, tier(中小→メガベンチャー→大企業→外資/GAFA),
            joinReq: { skills: {id,level}[] }, interviewer: 面接官monsterId,
            projects: 案件[](末尾にボス), salary(月給) }
  Freelance: 単発案件プール（プレイヤーLvでスケール・低報酬・所属不要）
  state.company: companyId | null   // null = フリーランス
  ```
- 既存の `案件(monsters)` はこの「単発プール」＋「各社ダンジョンの中身」として再編する。

## 0-10. 最初の大目標（3つのいずれかを達成）
3軸それぞれに“北”を置く。**どれか1つ達成で最初の章クリア**（＝起業/次周回のトリガ）。プレイスタイルで選べる:

| 軸 | 大目標 | 概要 |
|---|---|---|
| **金稼ぎ** | 🌙 **月（Moon）を購入** | 資産マイルストーンの最終到達点。とんでもない円のシンク＝富の証明。 |
| **案件** | 🏢 **でかい会社を制覇** | 最上位ティア企業のダンジョン（全案件＋ラスボス役員）をクリア。 |
| **技術** | 🧪 **新しい言語を開発** | 究極の技術プロジェクト。多数の言語を高Lvまで極め、特大の「言語開発」案件を完遂。 |

> 3つは並立。1つ達成すれば良いが、コンプ勢は全部狙える。達成内容で称号/周回ボーナスが変わる。
> 起業はこれらの達成を“前提条件”にはしない（[0-11①](#0-11-確定事項2026-06-16-決定)）が、達成してからの起業が本筋。

## 0-11. 確定事項（2026-06-16 決定）
1. **起業はいつでも可能（ランク/目標ゲート無し）**。ただし**力不足で起業すると多大なデバフ**、特に金銭面（生活費に対し収入源を失い**即・赤字地獄**）。賢い起業＝3大目標や土台を作ってから。準備不足の起業は罰。
   - 実装: `PRESTIGE_MIN_RANK` 撤廃。起業時の「準備度」でストック増減＆デバフを算出する仕組みを新設。
2. **「会社制覇」＝ダンジョン攻略であり、出世とは無関係**。出世は個人のキャリアの格で、**頂点は社長**（旧CTO表記を是正）。経営会議＝会社制覇の最終ボスであって出世昇進ではない。
   - 実装: `rank.ts` の頂点を社長に。会社制覇の判定は所属/出世と切り離す。
3. **生活費の赤字は全体に効く**：生産/制作/戦闘の**全効率ダウン＋メンタル上限↓**＋ショップ不可＋報酬天引き。深いほど重い。**詰まないがツラい**。①の“力不足起業デバフ”もこの赤字に直結。

### 0-11b. 知識の4軸と同時獲得（2026-06-17 決定）
技術知識を**4つの別軸**に分離。「Web/組み込みはドメイン(業界)か?」→ **No。あれは"何を作るか"の技術領域であって、業界ではない**。DDDで言うドメイン＝問題領域(業務)。

| 軸 | 問い | 中身 | 主な伸び口 |
|---|---|---|---|
| **言語** | どう書くか | JS/Python/… | コードを書く/概念/ライブラリ/OSS/資格 |
| **領域・プラットフォーム** | 何を作るか | Web/ゲーム/モバイル/組み込み/**AI・機械学習**/**データ基盤・分析** | **フレームワーク実装で副次取得**＋言語非依存のベース練習。※AIとデータは別物として分離 |
| **インフラ・基盤** | どこで動かすか | Linux/Docker/K8s/Terraform/AWS/GCP/**Azure**/Nginx/**データベース** | 言語非依存の独立ツール軸（橋渡し対象外）。**各スキルが実サービスを個別アクションで持つ**（言語の傘下ツリー並みの厚み） |
| **業界ドメイン** | 誰のために | 金融/EC・小売/医療/法務/公共 | 業務知識。言語にもプラットフォームにも依存しない。**将来は案件討伐で伸びる**（暫定は研究アクション） |

- **フレームワーク＝言語→領域の橋渡し**: 「React で実装」→ **JS(主) ＋ Web(副 約60%)**。Unity→C#＋ゲーム、PyTorch→Python＋AI・データ、Flutter→Dart＋モバイル、Arduino→C＋組み込み。
  - 実装: `GameAction.xpAlso?:{skill,xp}`。`techtree.ts` の各FWに `platform` を付与し `xpAlso:{skill:plat_${platform}}` を生成。`store.runSkillTick` が完了ごとに主＋副を付与（同XP補正）。SkillView がカードに「Web +XX」副次バッジ表示。
  - スキルid: 言語=`<id>`、領域=`plat_*`、インフラ=`inf_*`、業界=`dom_*`。サイドバーは CATEGORIES 順（言語→領域→インフラ→業界→クラフト）。
  - インフラ厚み(2026-06-17): `sectors.ts` の `INFRA_SPECS` に各スキルの実サービス一覧（AWS=EC2/S3/Lambda/RDS/DynamoDB/…13、GCP=BigQuery/GKE/…12、Azure=Cosmos DB/AKS/…11、DB=PostgreSQL/Redis/MongoDB/Kafka/…12 等）。各サービス=1アクション(産出 commit、Lv段階解禁)。「実サービスを中で経験して稼ぐ」。**DB はインフラ独立スキルとして復活**（SQL言語とは別レンズ）。
  - 一貫性ルール: **FWは"領域"だけに橋渡し**。ライブラリ/概念は言語純粋（pandas等データ系も暫定ライブラリ＝言語のみ。必要なら将来FW昇格で再検討）。**業界ドメインはFWでは伸びない**（言語/領域非依存だから）。紐づけ無しFW（Qt/Cocoa/.NET/Tokio/Akka/Hibernate＝汎用/デスクトップ/基盤）は言語のみ。
  - TODO(次の一手): 業界ドメインを**案件(モンスター)駆動**に。下記 0-11c。

### 0-11c. 「成果物(product)」廃止とリソース観の整理（2026-06-17 決定）
- **`product`(成果物) は概念ごと削除**。「何でもプロダクト」という汎用ブロブは意味不明でノイズだった。
  - 生産の単一リソースは **`commit`（仕事の成果単位）** に一本化。基礎アクション(コードを書く/概念/ライブラリ/領域学習/業界研究/インフラ構築)が `commit` を産む。
  - 上位アクション(フレームワーク/インフラ運用/領域構築/ドメイン分析)は **`commit` を消費して経験値に変換**（産出アイテム無し）。言語(主)＋領域(副)が同時に伸びる。
  - `commit` は**売却（金源）にも上位の素材にもなる**。物理制作物（基板/飯/PC）は通常アイテムで売れる＝出口があり「行き止まり」ではない。
  - **抽象的な成果はアイテム化しない**（2026-06-17 追加）。`product` と同じく `kudos`(感謝)・`cert`(資格)・`tech_debt`(技術的負債) も削除。OSS/資格アクションは **XPのみ**、仕様変更案件の負債ドロップも撤廃。原則: **知的活動が生むのは commit だけ。アイテムを生むのは物理制作のみ。資格/評価/負債は状態・効果・派生ステータスであってストレージに積む名詞ではない**（状態化は運用収入/入社条件の設計時に一緒に）。
  - クラフト(inputs を持つ)＝**物理制作のみ**になった: 料理(食材)/PC組立(パーツ)/はんだづけ(電子部品→基板)。知的生産は全て gather。
  - 実装: items から product 削除、`outputs:{}` 化、monster loot の product→commit、`progression.ts` にも `xpAlso` 反映（オフラインとliveの不一致を解消）、SAVE_VERSION 10。

### 0-11d. 業界ドメイン × 案件（要設計・次の一手）
ユーザー方針: **「ドメインは案件に繋がる。案件はどの業界の案件かを持ち、その案件レベルが高いほど有効」**。
- 案件(モンスター)に **業界(domain) と 案件レベル** を持たせる。
- 案件をこなすと対応する**業界ドメインXP**が入る（案件レベルが高いほど多い／高レベル案件ほど有効）。
- 逆に**業界ドメインレベルが、その業界の案件への有効度（命中/報酬/解禁）に効く**——という双方向。
- これにより 金融案件→金融ドメイン経験→（将来）金融会社の入社条件、へ接続（0-9）。
- ※現状の7案件は業界非依存の汎用エンジニア作業なので、**業界別の案件コンテンツ追加**が要る。これが次の実装対象。

### 0-11e. 運用収入＝放置収入（2026-06-17 決定）
放置収入の概念は要るが、**「リリースして成果物/サービスを生む」モデルは廃止**（新しい名詞=ノイズ）。
- **モノを作らない**。操作も持ち物も増えない。収入は **durable state から導出する純粋関数** にする。
- `mrr = f(主要スキルLv合計, 累積案件クリア, 役職/起業)` を tick/offline 同式で `gold += mrr*dt`。
- 品質＝実力。技術が高いほど運用収入が高い、で §0-5 の好循環が成立。
- 生活費(放置コスト)と相殺させて経済の緊張を作る（0-6）。
- ※「成果物」「サービス化」「リリース」という語・アイテム・アクションは SPEC から一掃済み。

## 1. コンセプト / 世界観

新卒の見習いエンジニアが、**見習い → ジュニア → … → 役員 → 社長** と成り上がる出世idle（頂点は社長）。
ダンジョンは「案件（プロジェクト）」、雑魚は「バグ」「仕様変更」、ボスは「上司」。
HPは肉体ではなく **メンタル（精神力）**。0になると死ぬのではなく **燃え尽き（バーンアウト）** して強制離脱。回復は飯ではなく **カフェイン・有給・スルー力**。

風刺コメディのトーンで、用語のあるあるネタを全面に出す（例: 「仕様が確定しました（確定していない）」）。

## 2. コアループ

```
学習/コーディングで【コミット】を生産
        ↓
フレームワーク実装で【コミット】を消費 → 言語＋領域の経験値に変換
        ↓
案件（プロジェクト）に出撃して納品＝戦闘 → 【円(給料)・経験・評価・業界ドメイン】
        ↓
評価で 昇進 & クラスチェンジ → 補正で全体が強くなる
        ↓ (放置中も自動で回る)
```

Melvorの「1度に1作業しかできず暇」を緩和するため、**部下（[§7](#7-部下システム)）** に別作業を並行で回させられる。

## 3. 役職ランク と クラスチェンジ補正 ★本作の肝

### 3.1 役職ランク（縦の出世軸）
総合レベル（または「評価ポイント」）で昇進。ランクが上がると全ステ倍率が上がり、上位の案件・職種が解禁。

出世は**個人のキャリアの格**であり、**会社制覇（ダンジョン攻略）とは無関係**（[0-11](#0-11-確定事項2026-06-16-決定)）。頂点は **社長**。

| ランク | 名称 | 解禁 |
|---|---|---|
| 1 | 見習い / インターン | 初期 |
| 2 | ジュニア | — |
| 3 | ミドル | 職種クラスチェンジ解禁 |
| 4 | シニア | 上位案件 |
| 5 | テックリード | 上位職 |
| 6 | 役員（VPoE/CTO） | 高難度案件 |
| 7 | **社長** | 出世の頂点 |

### 3.2 職種クラス（横の特化軸）= 補正
ミドル昇進時に **職種** を選択。職種ごとに **スキル/戦闘ステに乗算補正** がかかり、ビルドの個性になる。**転職（クラスチェンジ）可能**だが、再修行コストあり。

```ts
// data/classes.ts
interface JobClass {
  id: string; name: string; tier: number;
  modifiers: Modifier[];          // 乗算/加算補正
  passive?: string;               // 特殊効果
  upgradesFrom?: string;          // 上位職への分岐
}
```

| 職種 | 補正（例） | パッシブ |
|---|---|---|
| フロントエンド | UI実装 +30% / インフラ -10% | 見た目の評価+（顧客ウケ↑） |
| バックエンド | ロジック実装 +30% / デザイン -15% | 大型案件で被ダメ減 |
| インフラ/SRE | 環境構築 +40% / 実装速度 -10% | 障害（DoT）耐性 |
| データサイエンティスト | 分析 +35% | レアドロップ率↑ |
| QA | テスト +40% | 敵の「バグ」出現を抑制 |
| PM | 全スキル +10% / 個別ピーク低い | 部下の効率 +25% |
| **フルスタック**(上位) | 全実装 +15%（穴がない） | 並行案件スロット+1 |
| **セキュリティ**(上位) | 防御系 +50% | 「炎上」を確率で無効化 |

> 補正は既存の `modifiers` 機構（プログレス§ の倍率）で表現可能。クラス変更時に再計算するだけ。

## 4. スキル = 実際の tech stack（分野 → 言語 → フレームワーク）★改訂

スキルは「設計/開発」のような粗い括りではなく、**実在の技術スタック**を個別にレベリングする。
**分野(domain)** でグルーピングし、各分野に **言語(language)** と **フレームワーク(framework)** がある。
`SkillKind` は内部的に gather=言語 / craft=フレームワーク / combat=現場力 に対応。

### 4.1 グルーピング（Melvor風・折りたたみ）
サイドバーは **言語 / フレームワーク / 現場力 / その他** のセクションに分かれ、
各グループは折りたたみ可能（`data/groups.ts`、`Sidebar` の `open` state）。

**言語(30) — 系統別グループ:**
| グループ | 言語 |
|---|---|
| スクリプト系 | JavaScript, TypeScript, Python, Ruby, PHP, Lua, Perl |
| システム系 | C, C++, Rust, Go, Zig |
| JVM/.NET系 | Java, Kotlin, Scala, C# |
| ネイティブ/モバイル | Swift, Dart, Objective-C |
| 関数型 | Haskell, Elixir, Clojure, F#, Erlang |
| データ/科学 | R, Julia, SQL |
| レガシー | COBOL, Fortran, Assembly |

**フレームワークは独立スキルではなく「言語スキル配下のアクション」**（`data/techtree.ts`）。
1つの言語スキルが Melvor 的に多数のアクションを持ち、レベルで段階的に解禁される:

| カテゴリ | 例（JavaScript） | 産出 |
|---|---|---|
| 基礎 | JavaScript を書く (Lv1) | コミット |
| 概念 | クロージャ / 非同期・Promise / イベントループ | コミット |
| ライブラリ | jQuery / Lodash / Axios | コミット×2 |
| フレームワーク | Node.js / Express / Vue / Svelte | コミット消費→経験値(言語＋領域)に変換 |
| OSS | OSS にコミット (高Lv) | XPのみ（アイテム無し） |
| 資格 | JavaScript 認定 (JSNAD) (Lv35) | XPのみ（取得状態/効果は将来別途） |

> 全30言語が同様の配下ツリーを持つ（`techtree.ts` の `LANG_SPECS` に concepts/libraries/frameworks/oss/cert を書くと自動生成）。**解禁前のアクションも一覧表示し、ボタンは「Lv○○ で解禁（開始不可）」で無効化**。

### 4.2 ループ
- **言語を書く** → コミット産出、言語Lv↑。
- 概念/ライブラリ → さらにコミット、言語Lv↑（高効率だが高Lv解禁）。
- **フレームワークで実装** → コミット消費して **言語Lv＋領域Lv** に変換（産出アイテム無し）。
- OSS・資格 = 高Lvの到達目標（XPのみ。アイテムは生まない）。
- inputs を持つアクション＝「制作(craft)」補正。知的生産(言語/領域/インフラ/業界)は「生産(gather)」補正。物理制作(料理/PC組立/はんだづけ)のみ素材を消費。

### 4.3 現場力（combat ステ）
| id | 表示 | 意味 |
|---|---|---|
| debug | デバッグ力 | 命中(手数) |
| impl | 実装力 | 火力(max hit) |
| robust | 堅牢性 | 被害軽減 |
| mental | メンタル | 精神HP。0で燃え尽き |

回復 = カフェイン / 栄養ドリンク / 有給（大回復）。

### 4.4 アイコン ★改訂
**絵文字は使わない。** `react-icons`（Simple Icons のブランドロゴ + Lucide の概念アイコン）を
`ui/icons.tsx` のレジストリ経由で使用。データは `icon: "react"` のような文字列idのみ持つ。

## 5. 敵 = 案件・上司（戦闘デザイン）

「モンスター」を **案件(プロジェクト)** と **上司(ボス)** に置換。雑魚は案件内の障害。

### 5.0 編成: フリーランス単発プール ＋ 会社ダンジョン（[0-9](#0-9-世界モデル-案件モンスター--会社ダンジョン) 参照）
- **単発案件プール**（無所属で常時受注可・低報酬）: バグ / 新人教育 / 仕様変更 等の汎用案件。プレイヤーLvでスケール。
- **会社ダンジョン**（所属でアクセス・高報酬・固有）: 会社ごとに**順序付きの案件列＋末尾に役員/上司ボス**。全クリアで**制覇**。
  - 例: 中小SIer →（バグ修正→仕様変更対応→納期前デスマ→**課長**）。大企業/外資ほど長く・強く・報酬大。
- 下記の各案件タイプ（5.1/5.2 のギミック）は、単発プールにも会社ダンジョンの中身にも配置できる共通パーツ。
- **報酬は円＋現場力XPに加え、案件の `relatedSkills` に応じた言語/技術XP**（[0-9](#0-9-世界モデル-案件モンスター--会社ダンジョン)）。Web案件→React/TS、ゲーム案件→C#/Unity 等。

### 5.1 雑魚 / 障害（旧monster）
| 敵 | ネタ | ギミック |
|---|---|---|
| バグ | 基本の雑魚 | 普通に殴る |
| 仕様変更 | 倒す直前にHPが回復することがある | 「仕様が変わりました」 |
| 新人教育 | 与ダメ低いが時間がかかる | 倒すと部下候補ドロップ |
| レビュー指摘 | こちらの設計力で軽減 | 防御依存 |
| 本番障害(インシデント) | 継続ダメージ(DoT=メンタル削り) | 早期撃破推奨 |

### 5.2 ボス = 上司 / 面接官
| ボス | 特徴 |
|---|---|
| **面接官** | **入社ボス**。要件Lvを満たして挑戦→撃破で内定（所属＝ダンジョン解禁）。技術質問/ホワイトボードコーディング等のギミック。会社ティアほど強い |
| 課長 | 会社ダンジョンの最初のボス。残業要求でメンタル削り |
| 部長 | 「で、いつ終わるの？」で火力UP |
| 客先PM | 仕様変更を多重発生させる |
| 経営会議（最上位の役員ボス） | 全ギミック盛り。撃破で**でかい会社を制覇**（[3大目標](#0-10-最初の大目標3つのいずれかを達成)の1つ） |

### 5.3 メンタル & バーンアウト
- メンタル0 = 死亡ではなく **強制離脱（バーンアウト）**。資源ロスト無し、案件進捗だけリセット。
- 自動回復: メンタル50%以下で選択中のカフェイン/ドリンクを自動消費（既存の自動食事ロジック流用）。
- 「有給」= 高回復だが所持数が貴重（ショップ/評価報酬）。

> 既存 `Monster` 構造に `phases`（複数フェーズ）と `dot`（継続ダメ）を足すだけで大半が表現可能。

## 7. 部下システム ★Melvorの"暇"を解消

シニア昇進で **部下(ジュニア)** を採用可能。部下は自分とは別に作業を1つ持てる＝**並行運用**。

```ts
interface Subordinate {
  id: string; name: string;
  jobClass: string;      // 部下にも職種補正
  level: number;
  assignment: ActiveAction | null;  // 生産/制作/案件補助に割り当て
}
```

- 部下を **生産スキルに配置** → 放置中も素材が増える。
- 部下を **案件に同行** → 戦闘火力/メンタル肩代わり。
- 部下は「新人教育」案件や採用ショップで増える。育成・配置が **最適化パズル** になる。
- PM職は部下効率+25%（マネジメント特化ビルドが成立）。

> v1では未実装。エンジン的には「複数 activeAction の並行tick」への拡張が必要なので [§9](#9-将来拡張ロードマップ) 扱い。
